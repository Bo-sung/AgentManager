# Design handoff — 컴포저 "권한/안전 모드" 칩 (engine-aware)

> 클로드 디자인용 지시 프롬프트. 아래 전체를 그대로 전달 가능. 결과물은 기존 AgentManager
> 프로토타입 디자인 언어(다크 테마, 컴포저 하단 입력 영역)와 일관된 **HTML 프로토타입**.

---

## 배경
AgentManager는 여러 코딩 에이전트 CLI(cc=Claude Code, gx=Codex, agy=Antigravity, pi=Pi)를
한 곳에서 구동하는 Windows 관제 앱(WPF, 다크 테마, 3-pane). 하단 **컴포저**(입력창)에는 이미
모델 피커, 추론(reasoning) 피커, 번역 토글이 칩 형태로 있다.

여기에 **세션별 "권한/안전 모드"** 를 추가한다. 다른 에이전트 UX처럼 **현재 모드가 색으로
한눈에 보이고**, 입력창 안에서 바로 바꿀 수 있어야 한다.

## 핵심 원칙 — 엔진별 네이티브 모드 (통합 추상화 금지)
각 엔진의 권한 표면이 실제로 다르다. 가짜 통합 3-tier를 만들지 말고 **각 엔진의 네이티브
모드를 그대로** 노출한다. 단, **색(위험 그라데이션)만 공통**으로 묶어 일관성을 준다.

| 엔진 | 모드(왼→오 = 안전→위험) | 비고 |
|---|---|---|
| **cc** (Claude Code) | `Plan` · `Default(ask)` · `Bypass` | `--permission-mode plan` / broker / `--dangerously-skip-permissions` (Accept-edits는 phase 2) |
| **gx** (Codex) | `Read-only` · `Workspace-write` · `Full access` | `--sandbox` |
| **agy** (Antigravity) | `Full (auto)` — **고정, 변경 불가** | 항상 권한 스킵 |
| **pi** (Pi) | `Default` — **고정, 모드 없음** | 권한 개념 없음 |

- cc/gx: 클릭 시 메뉴(또는 사이클)로 모드 선택.
- agy/pi: **비활성 정적 배지**(드롭다운 없음). "왜 못 바꾸나" 혼동 없게 살짝 흐리게 + 툴팁.

## 공통 색 — 위험 그라데이션 (모드 이름 무관, 위험도로 색 결정)
- **안전/읽기 전용** (cc Plan, gx Read-only): 파랑/청록 계열(중립·안심).
- **작업/쓰기** (cc Default·Accept edits, gx Workspace-write): 초록(정상 작동).
- **전체 권한/우회** (cc Bypass, gx Full access, agy Full): 빨강/앰버(경고).
- pi Default(권한 없음): 무채색/중립 회색.

현재 모드의 색이 **칩 배경 또는 좌측 도트/보더**로 즉시 식별돼야 한다. (텍스트만으론 부족 —
"색으로 딱 보이게"가 목표.)

## 컴포넌트 요구사항
1. **위치**: 컴포저 툴바, 모델/추론 칩과 같은 줄(또는 인접). 공간이 좁으면 아이콘+짧은 라벨.
2. **닫힌 상태(기본)**: 작은 칩 — [방패/자물쇠 아이콘] + 모드명, 배경/보더가 위험 색.
3. **열린 상태**: 드롭다운/팝오버에 해당 엔진의 모드 목록. 각 항목에 이름 + 1줄 설명 + 위험 색
   도트. 현재 선택 표시(체크/하이라이트).
4. **agy/pi**: 칩이 비활성(클릭해도 안 열림), 약간 흐림 + 자물쇠 잠김 느낌, 툴팁으로 "이 엔진은
   모드 변경 불가" 안내.
5. **상태 전이**: 위험(빨강) 모드 선택 시 살짝 강조(경고 톤). 과하지 않게.
6. 다크 테마 토큰 사용(기존 프로토타입의 Bg/Txt/Accent/Warn/Err 계열과 일관).

## 산출물
- 컴포저 칩(닫힘) + 드롭다운(열림) + agy/pi 비활성 배지의 **HTML/CSS 프로토타입**.
- cc·gx·agy·pi 4개 엔진 각각의 상태를 한 화면에 비교로 보여주면 좋음.
- 색 토큰/상태 정의가 코드(WPF)로 옮기기 쉽게 명확히.

## 하지 말 것
- 통합 3-tier(ask/safe/yolo 같은) 명칭/구조로 묶지 말 것 — 엔진별 네이티브 유지.
- 모드 의미를 임의로 바꾸지 말 것(위 표가 진실, CLI 실측 기준).
- 컴포저 밖(설정 패널 등)으로 빼지 말 것 — **입력창 안에서 즉시 인지/변경**이 목표.
